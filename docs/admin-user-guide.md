## üë• **USER MANAGEMENT**

### **2.1 Viewing All Users**

1. **Navigate**: Go to Users ‚Üí All Users
2. **Filters**: Use filters to find specific users
3. **Search**: Search by name, email, or user ID
4. **Sort**: Sort by registration date, activity, or role

### **2.2 User Details and Profiles**

To view a user's profile:

1. **Click on User**: Click on any user from the user list
2. **Profile Information**: View complete user details
3. **Activity History**: See user's platform activity
4. **Booking History**: View all user bookings
5. **Support History**: See support tickets from this user

### **2.3 Suspending/Banning Users**

To suspend or ban a user:

1. **Select User**: Click on the user you want to suspend
2. **Actions Menu**: Click on "Actions" ‚Üí "Suspend User"
3. **Reason**: Provide a reason for suspension
4. **Duration**: Set suspension duration (temporary or permanent)
5. **Confirm**: Confirm the suspension

**Suspension Types:**
- **Temporary**: User can be reactivated after the period
- **Permanent**: User is permanently banned
- **Warning**: User receives a warning but remains active

### **2.4 Editing User Information**

To edit user information:

1. **Select User**: Click on the user
2. **Edit Button**: Click "Edit Profile"
3. **Modify Fields**: Update the required information
4. **Save Changes**: Click "Save Changes"
5. **Log Action**: The change will be logged in audit logs

### **2.5 Managing User Roles**

User roles include:

- **Customer**: Regular platform users
- **Provider**: Service providers (property owners, vehicle owners, tour guides)
- **Admin**: Platform administrators
- **Super Admin**: Full system access

To change user roles:

1. **Select User**: Click on the user
2. **Role Management**: Go to "Role Management"
3. **Select New Role**: Choose the appropriate role
4. **Confirm**: Confirm the role change

### **2.6 Handling User Disputes**

When handling user disputes:

1. **Review Complaint**: Read the full complaint details
2. **Gather Evidence**: Collect relevant information
3. **Contact Parties**: Reach out to both parties if needed
4. **Make Decision**: Decide based on platform policies
5. **Document Resolution**: Record the resolution in the system
